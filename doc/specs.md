# ТЗ: Birka — файловый менеджер для аудио и MIDI

## 1. Контекст и позиционирование
Birka — современный многовкладочный файловый менеджер, ориентированный на работу с аудио и MIDI, а также их метаданными. Целевая роль — замена стандартного файлового менеджера для профессионалов, работающих с аудио‑материалами и MIDI.

## 2. Бизнес‑цели
- Ускорить поиск и отбор медиа‑файлов без открытия DAW/редактора.
- Снизить время на рутину: переименование, классификация, предпросмотр.
- Повысить управляемость больших библиотек (10 000+ файлов).

## 3. Основные пользователи
- Саунд‑дизайнеры.
- Продюсеры и композиторы.
- Аранжировщики и MIDI‑программисты.

## 4. Функциональные требования

### 4.1 Метаданные
- Автоматическое чтение метаданных из аудио (BWF) и MIDI.
- Отображение метаданных в виде колонок.
- Фильтрация и сортировка по метаданным.

### 4.2 Предпросмотр
- Отображение волновых форм аудио.
- Структура MIDI‑файлов с зумом.
- Воспроизведение в один клик.

### 4.3 Многовкладочный интерфейс
- Вкладки для разных папок проекта.
- Сохранение пресетов вкладок.

### 4.4 Drag‑and‑Drop в DAW
- Перетаскивание отдельных дорожек из MIDI‑файла напрямую в секвенсор.

### 4.5 Пакетное переименование
- Массовое переименование с подстановкой метаданных.
- Поддержка шаблонов имени с BPM, Key, Date, Counter.

### 4.6 Пользовательские теги и рейтинги
- Пользовательские теги.
- Рейтинги.
- Комментарии и статусы.

### 4.7 Кастомизация контекстного меню
- Пользовательские действия для типов файлов и отдельных файлов.

### 4.8 Быстрый поиск и фильтрация
- Поиск по маске.
- Фильтр по метаданным.
- Фильтр по пользовательским тегам.

## 5. Нефункциональные требования
- Производительность: фильтрация библиотеки 10 000+ файлов за секунды.
- Надёжность: ошибки чтения метаданных не блокируют работу UI.
- Расширяемость: новые типы медиа должны добавляться без переписывания ядра.
- Удобство: минимальный “клик‑путь” к предпросмотру и фильтрации.

## 6. Платформа и форматы

| Параметр | Требование |
| --- | --- |
| Платформа | Windows 7/8/10/11 (Intel x86/x64, ARM64) |
| Формат приложения | Standalone (не плагин) |
| Поддерживаемые форматы | WAV (16/24-bit), AIFF, FLAC, MP3, OGG, MIDI |
| Лицензирование | Серийный номер |
| Демо‑версия | Полнофункциональна, без ограничений по времени |

## 7. Ограничения и допущения
- Birka — файловый менеджер, не аудио‑редактор.
- Полезность фильтрации зависит от качества метаданных в файлах.

## 8. Критические замечания (для QA)
- Проверить доступность только под Windows.
- Проверить устойчивость при отсутствующих/битых метаданных.
- Проверить корректность предпросмотра при больших аудио‑файлах и больших MIDI‑файлах.

## 9. Домашнее тестовое задание (acceptance)
- Открыть папку с библиотекой минимум 50 файлов.
- Настроить колонки BPM, Key, Duration, Rating.
- Переименовать 10 файлов по шаблону `[BPM]_[Key]_[OriginalName]`.
- Зафиксировать: ускорило ли Birka работу относительно стандартного менеджера.

## 10. Альтернативы для сравнения

| Инструмент | Платформа | Отличие |
| --- | --- | --- |
| Soundminer | Win/macOS | Проф. стандарт для аудио, высокая цена |
| BaseHead | Win/macOS | Сильные аудио‑метаданные, сложный UI |

## 11. Термины
- BPM: темп аудио.
- Key: тональность.
- Metadata: встроенная информация о файле.
- DAW: цифровая рабочая станция для аудио.

## 12. Не входит в область
- Аудио‑обработка и редактирование.
- Замена DAW.
